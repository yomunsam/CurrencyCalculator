# 架构决策记录（ADR）

## ADR-001：应用形态选择

- 决策：使用 Blazor WebAssembly + PWA，纯静态部署。
- 原因：满足无后端、可部署到 GitHub Pages/Cloudflare Pages、具备离线能力。

## ADR-002：汇率数据源策略

- 决策：采用三层策略：
  1) 实时 API（fawaz）
  2) 次级 API（Frankfurter，法币）
  3) 静态 fallback + localStorage 缓存
- 原因：在无需 API Key 的前提下提升可用性并减少单点故障。

## ADR-003：汇率基准模型

- 决策：内部统一使用 `USD -> X` 的汇率映射作为快照结构。
- 原因：多币种互转只需一次基准换算，计算逻辑稳定且易于扩展。

## ADR-004：状态持久化策略

- 决策：使用 `localStorage` 保存用户偏好与最近汇率缓存。
- 原因：无需后端即可实现跨会话体验与离线回退。

## ADR-005：国际化策略

- 决策：先采用轻量字典服务（en-US/zh-CN），后续按需升级为 .resx。
- 原因：MVP 阶段成本低、迭代快，且满足当前语言需求。
